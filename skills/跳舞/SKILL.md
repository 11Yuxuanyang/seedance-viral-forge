# 跳舞

本 skill 用来做“人物吸引力带场景种草”。重点是让观众想问“这是哪里”，不是低俗擦边。

共通底线见 [shared/SEEDANCE_BASELINE.md](/Users/yuxuanyang/ai-video/shared/SEEDANCE_BASELINE.md)。

## 目标

- 用人物亮相抓停留
- 用舞蹈走位带出商家环境
- 用场景向往感把流量引到门店

## 最小输入

- 商家类型
- 商家最值得拍的点
- 团购 / 套餐 / 卖什么

最好再补：

- 想要小姐姐、帅哥，还是不限
- 氛围关键词
- 重点想拍哪个区域

## 缺字段处理

- 缺角色偏好：默认“不限”，优先出最适配场景的人设
- 缺氛围：按商家类型推断
- 缺团购价格：不写价格逻辑，只做引流脚本
- 缺核心场景信息：先只问 1 个问题，优先补“店里最值得拍的是什么”

## 默认流程

1. 提取商家信息
2. 输出 3 个不同方向的方案卡
3. 等用户选 A/B/C，或说“都做”
4. 输出最终脚本

## 方案卡格式

```text
方案A｜[打法名]
角色：[甜妹 / 御姐 / 阳光男 / 西装男 / 其他]
舞感：[街舞 / 律动 / 慢拍 / 手势舞 / 轻爵士]
一句话画面：[人物、动作、场景三者同屏]
风险：[低 / 中 / 高 + 一句话]
```

规则：

- 三个方案的视觉风格不能重复
- 至少一个方案突出场景氛围
- 至少一个方案突出人物反差
- 场景必须能被看见，不能只是“在店里跳”

## 人设匹配原则

- 火锅、烧烤、夜宵：活力、烟火气、节奏感
- 咖啡、书店、甜品：清爽、轻盈、文艺
- 酒吧、酒店、夜店：高级、张力、慢拍
- 健身、运动、游泳：力量感、健康感

尺度控制：

- 可以有时尚感、镜头张力、律动感
- 不做低俗动作，不用露骨措辞
- 人物吸引力服务于场景，不反过来

默认只输出到对话，不自动保存文件。


## 写作硬规则

- 开场 2 秒必须有人物视觉锤
- 中段必须通过走位或转身带出场景核心区域
- 11-13 秒给最美定格或最佳角度
- 每次场景再出现，重复关键环境元素
- 每次人物再出现，重复关键外观特征
- 不要中途停下来口播介绍商家
- 结尾 CTA 要顺着情绪落地，比如“这家在哪”“周末去不去”

## 批量模式

只有用户明确说“批量”“矩阵”“多条”时启用。

先给角度矩阵，再分批生成。

矩阵四个维度：

- 区域：门口 / 大厅 / 包间 / 吧台 / 等位区
- 角色：甜妹 / 御姐 / 酷 girl / 阳光男 / 西装男
- 舞感：街舞 / 律动 / 手势舞 / 慢动作 / 轻爵士
- 情绪：欢快 / 高级 / 文艺 / 反差 / 松弛

## 保存策略

- 默认：只在对话输出
- 用户明确要求保存时，再按指定路径写入
- 若路径不明确，先确认建议路径
