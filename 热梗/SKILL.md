# 热梗

本 skill 只追一个指标：播放量。

共通底线见 [shared/SEEDANCE_BASELINE.md](/Users/yuxuanyang/ai-video/shared/SEEDANCE_BASELINE.md)。

## 目标

- 蹭到还热的东西
- 做出真人拍不了、但 AI 能做的画面
- 用已验证模式降低试错成本

## 两种启动方式

### 方式 1：用户没给热点

自动抓最近热点，再给 3 个方向。

搜索优先级：

1. 当天热搜
2. 近 72 小时热梗
3. 持续发酵的大事件
4. 已爆视频的同结构变体

### 方式 2：用户直接给热点

围绕指定热点直接出 3 个方向；如果用户还给了爆款链接或描述，则优先走“复刻结构”。

## 选题筛选门槛

每个候选方向都按 4 个维度打分，满分 10 分：

- 热度：0-3
- 视觉化空间：0-3
- 讨论潜力：0-2
- 模式匹配度：0-2

硬淘汰：

- 纯文字梗，画不出来
- 已经过气
- 明显政治敏感
- 负面社会事件、灾难、伤亡
- 只有模仿真人才成立，但无法用外貌特征替代

如果没有候选方向达到 7 分：

- 不硬生成脚本
- 直接告诉用户今天没必要做
- 改给 2 个更稳的替代方向

## 已验证模式

- 平行宇宙：如果 XX 是 XX
- 角色穿越：不属于这个场景的人物突然出现
- 名人对决：两个本不该同框的人 PK
- 热梗视觉化：把文本梗变成画面
- 爆款复刻：沿用已爆结构，换人物或环境

## 默认流程

1. 抓热点或解析用户给的热点
2. 输出 3 个不同模式的方案卡
3. 等用户选 A/B/C，或说“都做”
4. 输出最终提示词

## 方案卡格式

```text
方案A｜[模式名]｜评分[0-10]
热点：[蹭的是什么]
画面：[一句话，观众一眼能脑补]
互动点：[为什么会吵起来 / 转发 / 评论]
风险：[低 / 中 / 高 + 一句话]
```

规则：

- 三个方案尽量用不同模式
- 一句话画面必须具体，不要抽象词
- 不要长篇解释推理过程
- 涉及真人热点时，提示词里不用真实姓名，改写成外貌 / 身份特征；标题里可以写热点名

## 最终输出格式

默认只输出到对话，不自动保存文件。

```text
参数：竖屏9:16，[5-10秒]，[风格]

即梦提示词：
[一整段自然语言，不分 beat；写清场景、人物、动作、镜头、光影、声音]

标题：[≤55字，带 emoji]
话题：[10-12个标签]
元信息：模式 / 热点 / 预期讨论点
```

## 写作硬规则

- 热梗视频优先 5-10 秒，短就是优势
- 一个核心画面胜过复杂剧情
- 必须让人第一眼看懂“离谱点在哪”
- 提示词里写清具体环境、外观、动作顺序、镜头运动
- 不要写 logo、水印、品牌词
- 不要用英文提示词
- 不要直接复刻具体真人的脸和声音

## 复刻模式

用户给爆款链接或描述时：

1. 先提取原视频的结构
2. 保留结构，不照抄元素
3. 只替换人物、场景、梗
4. 明确说明新脚本的讨论点在哪里

## 发布时间建议

只给短建议，不要写运营长文：

- 热点：越快越好
- 非即时热梗：优先午休、下班后、睡前

## 保存策略

- 默认：只在对话输出
- 用户明确要求保存时，再按指定路径写入
- 若用户只说“保存一下”，先给建议目录再落盘
